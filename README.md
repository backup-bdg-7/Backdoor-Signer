# Backdoor Signer

## Project Structure

The codebase is organized into the following main directories:

- **App/** - Application setup and lifecycle
- **Core/** - Foundational services and utilities
- **Features/** - Feature-oriented modules
- **Magic/** - Low-level signing implementation
- **UI/** - Shared UI elements and design system

Each feature in the Features directory follows this structure:
- Models - Data structures
- Services - Business logic
- Views - UI controllers
- Components - Reusable UI elements

This organization improves maintainability, simplifies navigation, and makes adding new features easier.
